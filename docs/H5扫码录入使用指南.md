# H5 扫码录入功能使用指南

## 功能说明

H5 扫码录入功能允许老板通过手机扫描商品条形码或二维码，快速录入商品信息到库存系统。

---

## 快速开始

### 方式一：扫描二维码访问（推荐）

1. 用浏览器打开 `docs/扫码录入二维码.html` 文件
2. 用手机微信或浏览器扫描页面上的二维码
3. 自动跳转到扫码录入页面

### 方式二：直接输入网址

在手机浏览器输入：`https://bike.aipr0.com/scan`

---

## 使用步骤

### 1. 登录系统

首次访问需要登录，使用管理员或店主账号。

### 2. 进入扫码页面

- 从商品列表页点击 **"扫码录入"** 按钮
- 或直接访问 `/scan` 路径

### 3. 开始扫码

1. 点击 **"开始扫码"** 按钮
2. 浏览器会请求摄像头权限，点击 **"允许"**
3. 将手机摄像头对准商品条形码或二维码
4. 扫描成功后，条形码会自动填充到表单

### 4. 填写商品信息

扫码成功后，填写以下信息：

| 字段 | 是否必填 | 说明 |
|---|---|---|
| 条形码/二维码 | 必填 | 扫码自动填充，也可手动输入 |
| 商品名称 | 必填 | 例如：捷安特 ATX 810 |
| 品牌 | 必填 | 例如：捷安特 |
| 型号 | 必填 | 例如：ATX 810 |
| 分类 | 必填 | 从下拉列表选择 |
| 预警阈值 | 可选 | 库存低于此值时预警 |
| 质保期(月) | 可选 | 质保月数 |

### 5. 提交保存

点击 **"提交"** 按钮完成录入。

## 注意事项

### ⚠️ HTTPS 要求

**摄像头 API 必须在 HTTPS 环境下才能使用**，否则浏览器会拒绝访问摄像头。

| 访问方式 | 是否可用 |
|---|---|
| `https://bike.aipr0.com` | ✅ 可用 |
| `http://192.168.1.x` | ❌ 不可用 |
| `localhost` | ✅ 可用（仅限本地开发） |

### 📱 浏览器兼容性

推荐使用以下浏览器：

- ✅ Chrome（Android/iOS）
- ✅ Safari（iOS）
- ✅ Edge（Android）
- ✅ 微信内置浏览器
- ⚠️ 部分国产浏览器可能不支持

### 🔐 权限设置

首次使用时，浏览器会请求摄像头权限：

1. iOS Safari：点击 **"允许"**
2. Android Chrome：点击 **"允许"**
3. 微信浏览器：点击 **"允许"**

如果误点了 **"拒绝"**，需要在浏览器设置中手动开启摄像头权限。

### 🌐 网络要求

- 需要连接互联网（WiFi 或移动数据）
- 确保能访问 `bike.aipr0.com`

## 常见问题

### Q1: 点击"开始扫码"后没有反应？

**原因**：摄像头权限被拒绝

**解决**：
1. 检查浏览器地址栏是否有摄像头图标
2. 点击图标，选择 **"允许"**
3. 刷新页面重试

### Q2: 提示"无法启动摄像头"？

**原因**：
- 使用了 HTTP 而不是 HTTPS
- 浏览器不支持摄像头 API
- 摄像头被其他应用占用

**解决**：
1. 确认访问的是 `https://bike.aipr0.com/scan`
2. 更换浏览器（推荐 Chrome 或 Safari）
3. 关闭其他使用摄像头的应用

### Q3: 扫码识别不出来？

**原因**：
- 光线不足
- 条形码模糊或损坏
- 距离太近或太远

**解决**：
1. 在光线充足的地方扫描
2. 保持 10-20cm 的距离
3. 保持手机稳定
4. 如果实在扫不出，可以手动输入条形码

### Q4: 扫码成功但提交失败？

**原因**：
- 网络连接中断
- 必填字段未填写
- 服务器错误

**解决**：
1. 检查网络连接
2. 确认所有必填字段已填写
3. 查看错误提示信息

### Q5: 首次访问提示"您的连接不是私密连接"？

**原因**：使用了自签证书或 Cloudflare 源站证书

**解决**：
1. 点击 **"高级"** 或 **"详细信息"**
2. 点击 **"继续访问"** 或 **"访问此网站"**
3. 后续访问不会再提示

## 技术说明

### 使用的技术栈

- **html5-qrcode**：开源扫码库，支持条形码和二维码
- **React + Ant Design**：前端框架和 UI 组件
- **UMI**：企业级前端框架

### 支持的码制

- 一维码：EAN-13、EAN-8、UPC-A、UPC-E、Code 128、Code 39 等
- 二维码：QR Code

### 页面特性

- 📱 移动端优化，响应式布局
- 🎯 自动对焦，快速识别
- 🔄 支持手动输入条形码
- ✅ 表单验证，防止错误提交
- 🚀 无需安装 App，浏览器直接使用

## 开发相关

### 本地开发

```bash
cd frontend
pnpm install
pnpm start
```

访问：`http://localhost:8000/scan`

### 文件位置

- 页面组件：`frontend/src/pages/scan/index.tsx`
- 样式文件：`frontend/src/pages/scan/index.less`
- 路由配置：`frontend/config/routes.ts`

### 部署

```bash
cd frontend
pnpm build
```

构建产物在 `frontend/dist` 目录。
