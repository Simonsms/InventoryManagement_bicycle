# 商品管理模块使用说明

## 1. 模块定位

商品管理用于维护系统中的标准商品主数据，是库存、入库、调拨、盘点、预警等业务的基础。

当前系统里的核心原则是：

- 先有商品，再做库存业务
- 商品停用后，不再用于新业务，但历史单据仍保留
- 商品信息按“分类 + 品牌 + 名称 + 型号”组织，供后续库存和导入流程引用

## 2. 适用场景

- 新增一个可销售或可管理的自行车商品
- 修正商品所属分类、品牌、型号、质保期、预警阈值
- 停用已不再经营的旧商品，避免继续被录入库存业务

不适用的场景：

- 批量创建商品：请走“批量导入”
- 修改库存数量：请走库存、入库、调拨、盘点等业务模块
- 删除历史商品及关联数据：当前模块不提供物理删除

## 3. 入口与菜单

当前左侧导航顺序已调整为：

`仪表盘 -> 商品管理 -> 批量导入 -> 库存管理 -> 出入库流水 -> 调拨管理 -> 盘点管理 -> 预警中心 -> 系统设置`

这样做的目的是让用户登录后优先维护商品主数据，再进入后续库存业务。

## 4. 权限说明

### 4.1 列表查看

- 已登录用户可查看商品列表

### 4.2 新增与编辑

- 店主 `shop_owner`
- 门店管理员 `store_admin`

### 4.3 停用商品

- 仅店主 `shop_owner`

说明：

- 停用是软删除，不会清除历史库存、流水、调拨、盘点记录
- 停用后商品会从“有效商品”列表中消失，不能再用于新的业务录入

## 5. 页面功能

当前页面位于：

- 前端页面：[index.tsx](/E:/InventoryManagement_bicycle/frontend/src/pages/products/index.tsx)
- 后端接口：[products.ts](/E:/InventoryManagement_bicycle/backend/src/routes/products.ts)

### 5.1 商品列表

列表当前展示以下字段：

- 商品名称
- 品牌
- 型号
- 分类
- 预警阈值
- 质保（月）

### 5.2 操作按钮

- `新增商品`
- `编辑`
- `停用`

其中：

- `新增商品`、`编辑` 仅管理员/店主可见
- `停用` 仅店主可见

### 5.3 新增 / 编辑表单

当前页面表单支持维护以下字段：

- 分类：必填
- 商品名称：必填
- 品牌：必填
- 型号：选填
- 质保期（月）：选填
- 低库存预警阈值：选填，默认 `2`

## 6. 字段规则

### 6.1 必填字段

- 分类
- 商品名称
- 品牌

### 6.2 默认值

- `lowStockThreshold` 默认值为 `2`

### 6.3 当前未在页面开放的字段

后端商品实体中还存在 `specs` 字段，用于承载 JSON 规格信息，但当前商品管理页面没有提供录入和编辑入口。

这意味着：

- 通过页面新增/编辑商品时，无法直接维护规格 JSON
- 如果确实需要维护规格，只能通过后端接口或批量导入间接写入

## 7. 接口行为

### 7.1 查询商品列表

- `GET /api/v1/products`

后端支持以下过滤参数：

- `category_id`
- `brand`
- `keyword`

说明：

- `keyword` 当前匹配商品名称或型号
- 列表只返回 `is_active = true` 的商品

### 7.2 新增商品

- `POST /api/v1/products`

请求体核心字段：

- `categoryId`
- `name`
- `brand`
- `modelNumber`
- `specs`
- `warrantyMonths`
- `lowStockThreshold`

### 7.3 编辑商品

- `PUT /api/v1/products/:id`

允许按字段局部更新。

### 7.4 停用商品

- `DELETE /api/v1/products/:id`

真实行为是把 `isActive` 置为 `false`，不是物理删除。

## 8. 使用建议

建议按下面顺序使用本模块：

1. 先确认分类已存在；如果分类还没建，可先走批量导入自动建类，或补建分类
2. 新商品优先在商品管理中建立主数据
3. 需要批量建档时再走批量导入
4. 商品建好后，再进行批量入库、库存调整、调拨、盘点

## 9. 当前边界与限制

以下能力当前代码中未实现或未开放：

- 没有商品导出
- 没有批量停用
- 没有图片、条码、售价等扩展字段
- 没有页面级规格 JSON 维护入口
- 没有物理删除

如果后续要继续补文档或做功能扩展，优先级建议是：

1. 明确商品唯一性规则，避免重复建档
2. 补规格字段维护方式
3. 补导入模板下载与示例文件
