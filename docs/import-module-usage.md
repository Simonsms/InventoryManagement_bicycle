# 批量导入模块使用说明

## 1. 模块定位

批量导入用于把 Excel / CSV 文件中的数据一次性写入系统，减少人工逐条录入。

当前模块包含两类导入：

- `批量导入商品`
- `批量入库`

对应页面位于：

- 前端页面：[index.tsx](/E:/InventoryManagement_bicycle/frontend/src/pages/import/index.tsx)
- 后端接口：[import.ts](/E:/InventoryManagement_bicycle/backend/src/routes/import.ts)

## 2. 适用场景

### 2.1 批量导入商品

适用于：

- 系统初始上线时一次性建立商品档案
- 新品牌、新车型集中上架
- 供应商一次性提供了商品基础资料表

### 2.2 批量入库

适用于：

- 新店开业时，集中导入首批库存
- 总部或仓库线下整理好进货表后，批量补录库存
- 同一门店一次性录入多条采购入库数据

不适用的场景：

- 门店之间调货：请走调拨管理
- 单条人工入库：请走库存管理里的入库操作
- 盘点差异修正：请走盘点管理

## 3. 入口与权限

当前菜单入口在：

- 左侧导航 `批量导入`

访问与执行权限：

- 店主 `shop_owner`
- 门店管理员 `store_admin`

说明：

- 普通店员当前没有导入权限
- 页面本身受前端访问控制约束，接口也要求登录并校验角色

## 4. 页面实际能力

页面当前提供两张导入卡片：

- `批量导入商品`
- `批量入库`

每张卡片都支持：

- 拖拽或点击上传文件
- 展示成功条数
- 展示失败条数
- 展示逐行错误信息

支持上传格式：

- `.xlsx`
- `.xls`
- `.csv`

默认行为：

- 后端只读取文件中的第一个工作表

## 5. 批量导入商品

### 5.1 模板字段

当前代码支持以下表头：

- `分类名称` 或 `category`
- `品牌` 或 `brand`
- `商品名称` 或 `name`
- `型号` 或 `modelNumber`
- `规格` 或 `specs`
- `质保月数` 或 `warrantyMonths`
- `预警阈值` 或 `lowStockThreshold`

其中必填项是：

- 分类名称
- 品牌
- 商品名称

### 5.2 实际导入行为

- 如果分类不存在，系统会自动创建分类
- 每一行会直接创建一条新商品
- `低库存预警阈值` 为空时，默认写入 `2`
- `规格` 支持 JSON 字符串；如果 JSON 解析失败，系统会回退为空对象 `{}`

### 5.3 重要限制

- 当前没有去重逻辑
- 同一个商品重复导入，会重复创建商品记录
- 当前没有“预览后确认”步骤，上传后立即执行导入
- 当前没有模板下载按钮，页面只展示模板列说明

## 6. 批量入库

### 6.1 模板字段

当前代码支持以下表头：

- `商品名称` 或 `productName`
- `品牌` 或 `brand`
- `数量` 或 `quantity`
- `进货日期` 或 `purchaseDate`
- `成本价` 或 `costPrice`
- `批次号` 或 `batchNo`

其中必填项是：

- 商品名称
- 品牌
- 数量
- 进货日期

### 6.2 门店选择规则

- 店主 `shop_owner` 导入库存时，必须先选择门店
- 门店管理员 `store_admin` 不需要选门店，系统直接使用其绑定门店

如果店主未选择门店，前端会直接提示错误，不允许上传。

### 6.3 实际导入行为

每导入一行，系统会按事务执行以下动作：

1. 校验商品是否存在且处于启用状态
2. 校验数量必须大于 `0`
3. 查找或创建该门店下的商品库存汇总
4. 增加库存汇总数量
5. 创建一条库存批次
6. 记录一条 `StockMovement`，备注为 `Excel 批量导入`

额外规则：

- 如果未填写 `批次号`，系统会自动生成批次号
- 如果商品不存在，该行会失败
- 如果数量小于等于 `0`，该行会失败

### 6.4 重要限制

- 当前没有幂等控制
- 同一个文件重复导入，会重复增加库存
- 当前没有“先预览再确认”的缓冲阶段
- 当前没有导入回滚页面，错误只能通过后续业务修正

## 7. 返回结果说明

无论是商品导入还是批量入库，接口都会返回统一结果结构：

- `success`：成功条数
- `failed`：失败条数
- `errors`：错误信息数组

页面会直接把失败行错误展示在卡片下方。

## 8. 使用建议

推荐按下面顺序操作：

1. 先导入商品，再导入库存
2. 导入前先抽样检查表头和日期格式
3. 首次导入时先拿 1 到 3 行做小样验证
4. 确认结果正确后，再导入完整文件
5. 大批量入库时，避免重复上传同一文件

## 9. 常见失败原因

### 9.1 商品导入失败

常见原因：

- 缺少分类名称
- 缺少品牌
- 缺少商品名称
- 单行数据格式异常

### 9.2 批量入库失败

常见原因：

- 缺少商品名称、品牌、数量、进货日期
- 对应商品尚未建档
- 数量小于等于 `0`
- 店主未选择门店
- 当前用户未绑定门店

## 10. 当前边界与后续建议

当前代码里还没有以下能力：

- 模板下载
- 导入预览
- 重复文件识别
- 导入撤销
- 导入任务历史

后续如果要继续提升这个模块，建议优先补：

1. 模板下载与示例文件
2. 预览校验页
3. 商品唯一性校验
4. 导入幂等控制
5. 导入任务审计记录
